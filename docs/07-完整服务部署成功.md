## 部署步骤

### 1. MySQL数据库安装 ✅

```bash
# 安装MySQL
apt-get install -y mysql-server mysql-client

# 创建数据库和用户
CREATE DATABASE IF NOT EXISTS ky_project CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'ky_user'@'localhost' IDENTIFIED BY 'ky_password';
GRANT ALL PRIVILEGES ON ky_project.* TO 'ky_user'@'localhost';
FLUSH PRIVILEGES;

# 初始化数据库schema
mysql -u ky_user -pky_password ky_project < /tmp/init-db.sql
```

**数据库表结构**:
- `devices` - 设备信息表
- `command_records` - 指令记录表
- `device_data` - 设备数据表
- `alerts` - 告警记录表
- `sessions` - 会话表

**测试数据**: 3个设备（DEV-001, DEV-002, DEV-003）

### 2. 后端服务启动 ✅

**发现并修复的问题**:

#### 问题1: 日志级别大小写不匹配
- **现象**: 日志无法输出到控制台
- **原因**: 配置文件中`LOG_LEVEL=INFO`（大写），但winston需要小写`info`
- **修复**: 在`src/utils/logger.ts`中添加`.toLowerCase()`转换
```typescript
level: config.log.level.toLowerCase(),
```

#### 问题2: WebSocket服务器启动时机不当
- **现象**: 5000端口未监听
- **原因**: WebSocket服务器在TCP设备连接之后才启动，导致连接等待
- **修复**: 将`wsServer.start()`移到TCP设备连接之前

### 3. 前端服务启动 ✅

前端服务已成功启动，运行在3000端口。

## 已知限制

### 1. TCP设备连接 ⚠️

**现象**: 数据库中的设备（DEV-001, DEV-002, DEV-003）连接超时

**原因**: 这些设备的IP地址（192.168.1.101/102/103）在测试环境中不可达

**影响**: 不影响WebSocket通信和前端功能，但无法接收真实设备的TCP数据

**解决方案**:
- 方案A: 部署到包含真实设备的网络环境
- 方案B: 使用模拟TCP服务器进行测试
- 方案C: 修改数据库中的设备地址为可访问的地址

### 2. Redis未配置 ⚠️

**现象**: Redis连接未测试

**影响**: 可能影响某些缓存功能，但不影响基本通信

**解决方案**: 如果需要Redis功能，安装并启动Redis服务器

## 故障排除

### 问题: 后端无法启动

**检查清单**:
1. MySQL是否运行: `systemctl status mysql`
2. 数据库连接是否正常: `mysql -u ky_user -pky_password ky_project`
3. 端口5000是否被占用: `ss -tuln | grep 5000`
4. 查看日志: `tail -f proxy-server/logs/error.log`

### 问题: WebSocket连接失败

**检查清单**:
1. 后端服务是否运行: `ps aux | grep node.*dist/index.js`
2. 端口5000是否监听: `ss -tuln | grep 5000`
3. 使用wscat测试: `wscat -c ws://localhost:5000/ws`

### 问题: 前端无法连接后端

**检查清单**:
1. 检查前端WebSocket URL配置
2. 浏览器控制台检查错误信息
3. 网络连接检查

## 一、项目概述

### 1.1 项目定位

KY项目是一个设备管控与数据通信系统，通过Node.js代理服务实现前端与多个外部设备之间的高效、稳定通信。

### 1.2 核心目标

- 实现前端Vue3与Node.js代理服务的WebSocket双向通信
- 代理服务通过TCP与多个外部设备通信
- 支持数千级并发连接（3000-5000）
- 数据延迟控制在1秒以内
- 实现自动重连、心跳检测等高可用机制

### 1.3 技术路线

```
前端(Vue3) ←─WebSocket─→ Node.js代理 ←─TCP─→ 外部设备
```

### 2.2 数据流向

#### 前端发送指令流程

```
Vue3前端
  → WebSocket消息
  → Node.js代理(接收)
  → 消息路由(识别目标设备)
  → TCP连接池(获取设备连接)
  → 外部设备(执行指令)
  ← 设备响应
  ← TCP连接池
  ← 消息路由
  ← WebSocket
  ← 前端展示
```

#### 设备上报数据流程

```
外部设备(产生数据)
  → TCP服务
  → Node.js代理(TCP客户端接收)
  → 消息路由(查找订阅者)
  → WebSocket(广播给订阅的前端)
  → Vue3前端(更新UI)
```

#### 架构特点
- 前端直接连接Node.js代理服务，无中间代理层
- Node.js单实例处理所有WebSocket和TCP连接
- 简化架构，降低部署复杂度
- 适用于中小规模场景（单设备部署或局域网场景）

## 五、高并发与性能设计

### 5.1 并发能力设计

**目标并发**：1000-2000并发连接（单实例）

| 层级 | 并发处理方案 |
|------|-------------|
| 前端WebSocket | Node.js Event Loop + 异步非阻塞I/O |
| TCP连接池 | 连接复用 + 心跳保活 |
| 数据库 | 连接池(50) + Redis缓存 |
| 内存 | Redis缓存热点数据（设备状态、会话信息） |

### 5.2 低延迟保障

**目标延迟**：<1秒

| 延迟来源 | 优化方案 |
|---------|---------|
| 网络传输 | WebSocket长连接，减少握手开销 |
| 消息处理 | 异步非阻塞I/O，事件驱动架构 |
| 数据库 | 索引优化 + 缓存 + 批量写入 |
| TCP通信 | Keep-Alive保活 + 连接复用 |

### 5.3 性能指标

| 指标 | 目标值 | 监控方式 |
|------|--------|---------|
| WebSocket连接数 | <2000 | 日志监控 |
| 消息吞吐量 | >500 msg/s | 日志统计 |
| 前端响应时间 | <500ms | 前端埋点 |
| 设备响应时间 | <800ms | 后端日志 |
| 连接成功率 | >99.5% | 日志统计 |
| 错误率 | <0.5% | 错误日志 |

## 六、可靠性设计

### 6.1 自动重连机制

#### 前端重连
- 指数退避策略：3s → 6s → 12s → 24s → 48s
- 最大重连次数：5次
- 断线后自动重连，无需用户干预

#### 后端重连（TCP）
- 设备断线后5秒重试
- 连续失败10次后标记为"异常状态"
- 支持手动触发重连

### 6.2 心跳检测

| 层级 | 心跳间隔 | 超时判定 |
|------|---------|---------|
| 前端-代理 | 30秒 | 60秒无响应 |
| 代理-设备 | 60秒 | 120秒无响应 |

### 6.3 异常处理

- **网络波动**：消息队列缓存，网络恢复后重发
- **设备离线**：前端实时显示离线状态，禁止发送指令
- **服务重启**：前端自动重连机制，重新建立WebSocket连接
- **数据库故障**：降级到内存缓存，故障恢复后同步

## 二、数据库配置

### 2.1 MySQL 配置

#### 安装 MySQL

```bash
sudo apt update
sudo apt install -y mysql-server
sudo mysql_secure_installation
```

#### 创建数据库

```sql
-- 登录MySQL
sudo mysql -u root -p

-- 创建数据库
CREATE DATABASE ky_project CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'ky_user'@'%' IDENTIFIED BY 'your_password';

-- 授权
GRANT ALL PRIVILEGES ON ky_project.* TO 'ky_user'@'%';
FLUSH PRIVILEGES;
```

#### 导入初始表结构

```bash
mysql -u ky_user -p ky_project < database/schema.sql
```

#### 配置优化

编辑 `/etc/mysql/mysql.conf.d/mysqld.cnf`：

```ini
[mysqld]
# 连接数配置
max_connections = 500
max_connect_errors = 10000

# 缓存配置
innodb_buffer_pool_size = 2G
innodb_log_file_size = 512M

# 查询缓存
query_cache_size = 128M
query_cache_type = 1

# 慢查询日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow-query.log
long_query_time = 2
```

重启 MySQL：

```bash
sudo systemctl restart mysql
sudo systemctl enable mysql
```

---

### 2.2 Redis 配置

#### 安装 Redis

```bash
sudo apt install -y redis-server
```

#### 配置 Redis

编辑 `/etc/redis/redis.conf`：

```ini
# 绑定地址
bind 127.0.0.1

# 端口
port 6379

# 密码（生产环境必须设置）
requirepass your_redis_password

# 持久化
save 900 1
save 300 10
save 60 10000

# 内存限制
maxmemory 1gb
maxmemory-policy allkeys-lru
```

重启 Redis：

```bash
sudo systemctl restart redis
sudo systemctl enable redis
```

## 三、后端部署

### 3.1 安装依赖

```bash
cd proxy-server
pnpm install
```

### 3.2 配置环境变量

创建 `.env` 文件：

```env
NODE_ENV=production

# WebSocket配置
WS_PORT=5000
WS_PATH=/ws
WS_MAX_CONNECTIONS=5000
WS_HEARTBEAT_INTERVAL=30000
WS_HEARTBEAT_TIMEOUT=60000

# TCP配置
TCP_RECONNECT_INTERVAL=5000
TCP_HEARTBEAT_INTERVAL=60000

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ky_project
DB_USER=ky_user
DB_PASSWORD=your_password
DB_POOL_MAX=100
DB_POOL_MIN=10

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password
REDIS_DB=0

# JWT配置
JWT_SECRET=your-secret-key-change-in-production
JWT_EXPIRES_IN=24h

# 日志配置
LOG_LEVEL=info
LOG_FILE_PATH=./logs
```

### 3.3 构建项目

```bash
pnpm build
```

### 3.4 使用 PM2 启动

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 3.5 验证服务

```bash
# 查看状态

# 查看日志
pm2 logs ky-proxy

# 测试WebSocket
wscat -c ws://localhost:5000/ws
```

## 五、Nginx 配置

### 5.1 创建配置文件

创建 `/etc/nginx/sites-available/ky-proxy`：

```nginx
upstream websocket_backend {
    least_conn;
    server 127.0.0.1:5000;
    keepalive 64;
}

upstream api_backend {
    least_conn;
    server 127.0.0.1:5000;
    keepalive 64;
}

server {
    listen 80;
    server_name your-domain.com;

    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 前端静态文件
    location / {
        root /var/www/ky-frontend;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 缓存配置
        expires 1h;
        add_header Cache-Control "public, immutable";
    }

    # WebSocket代理
    location /ws {
        proxy_pass http://websocket_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置
        proxy_connect_timeout 7d;
        proxy_send_timeout 7d;
        proxy_read_timeout 7d;
        
        # Buffer配置
        proxy_buffering off;
        proxy_request_buffering off;
        
        # WebSocket特定头部
        proxy_set_header X-WebSocket-Server "ky-proxy";
    }

    # API代理
    location /api {
        proxy_pass http://api_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }

    # 健康检查
    location /health {
        proxy_pass http://api_backend/health;
        access_log off;
    }

    # 日志配置
    access_log /var/log/nginx/ky-proxy-access.log;
    error_log /var/log/nginx/ky-proxy-error.log;
}
```

### 5.2 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/ky-proxy /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 九、备份与恢复

### 9.1 数据库备份

```bash
# 创建备份
mysqldump -u ky_user -p ky_project > backup_$(date +%Y%m%d_%H%M%S).sql

# 恢复备份
mysql -u ky_user -p ky_project < backup_20240101_120000.sql
```

### 9.2 定时备份

创建 `/etc/cron.daily/ky-backup`：

```bash
#!/bin/bash
BACKUP_DIR="/var/backups/ky"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u ky_user -p'your_password' ky_project > $BACKUP_DIR/db_$DATE.sql

# 备份Redis
redis-cli -a 'your_redis_password' --rdb $BACKUP_DIR/redis_$DATE.rdb

# 删除30天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
find $BACKUP_DIR -name "*.rdb" -mtime +30 -delete
```

设置权限：

```bash
sudo chmod +x /etc/cron.daily/ky-backup
```

## 十、故障排查

### 10.1 常见问题

#### 问题1：WebSocket连接失败

**症状**：前端无法连接到WebSocket服务

**排查步骤**：

```bash
# 检查服务是否运行
pm2 status

# 检查端口是否监听
ss -tuln | grep 5000

# 检查防火墙
sudo ufw status

# 查看应用日志
pm2 logs ky-proxy
```

**解决方案**：

```bash
# 重启服务
pm2 restart ky-proxy

# 检查Nginx配置
sudo nginx -t
sudo systemctl reload nginx
```

---

#### 问题2：设备连接失败

**症状**：代理服务无法连接到TCP设备

**排查步骤**：

```bash
# 检查网络连通性
ping 192.168.1.100
telnet 192.168.1.100 8080

# 查看应用日志
pm2 logs ky-proxy | grep TCP
```

**解决方案**：

- 检查设备网络配置
- 检查防火墙规则
- 验证设备IP和端口是否正确

---

#### 问题3：数据库连接失败

**症状**：应用无法连接到MySQL

**排查步骤**：

```bash
# 检查MySQL状态
sudo systemctl status mysql

# 测试连接
mysql -u ky_user -p -h localhost

# 查看错误日志
sudo tail -f /var/log/mysql/error.log
```

**解决方案**：

```bash
# 重启MySQL
sudo systemctl restart mysql

# 检查用户权限
mysql -u root -p
> GRANT ALL PRIVILEGES ON ky_project.* TO 'ky_user'@'%';
> FLUSH PRIVILEGES;
```

---

#### 问题4：内存占用过高

**症状**：Node.js进程内存持续增长

**排查步骤**：

```bash
# 查看内存使用
pm2 monit

# 查看详细内存
pm2 show ky-proxy

# 生成内存快照
pm2 profile ky-proxy --depth 10
```

**解决方案**：

- 检查是否有内存泄漏
- 重启服务释放内存
- 优化代码逻辑

## 十一、性能优化

### 11.1 Nginx 优化

编辑 `/etc/nginx/nginx.conf`：

```nginx
worker_processes auto;
worker_connections 4096;

http {
    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    
    # 缓存配置
    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g inactive=60m;
    
    # 连接池
    upstream websocket_backend {
        server 127.0.0.1:5000;
        keepalive 64;
    }
}
```

### 11.2 MySQL 优化

```sql
-- 分析慢查询
```

## 十二、升级与维护

### 12.1 应用升级

```bash
# 1. 备份数据库
mysqldump -u ky_user -p ky_project > backup_before_upgrade.sql

# 2. 停止服务
pm2 stop ky-proxy

# 3. 拉取新代码
git pull origin main

# 4. 安装依赖
pnpm install

# 5. 构建
pnpm build

# 6. 运行数据库迁移
pnpm sequelize db:migrate

# 7. 启动服务
pm2 start ky-proxy

# 8. 验证
pm2 logs ky-proxy
```

### 12.2 定期维护任务

创建 `/etc/cron.weekly/ky-maintenance`：

```bash
#!/bin/bash

# 清理日志文件
find /var/log -name "*.log" -size +100M -exec truncate -s 0 {} \;

# 优化数据库
mysql -u ky_user -p'your_password' -e "OPTIMIZE TABLE ky_project.devices; OPTIMIZE TABLE ky_project.command_records;"

# 清理过期告警
mysql -u ky_user -p'your_password' -e "DELETE FROM ky_project.alerts WHERE resolved = TRUE AND created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);"

# 清理过期数据
mysql -u ky_user -p'your_password' -e "DELETE FROM ky_project.device_data WHERE received_at < DATE_SUB(NOW(), INTERVAL 90 DAY);"
```

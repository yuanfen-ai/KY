## 概述

已成功配置KY项目以支持在沙箱环境中预览，前端服务运行在5000端口（沙箱标准端口），后端WebSocket服务运行在8080端口。

## 服务配置

### 端口映射

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端（Vite） | 5000 | 沙箱预览端口，可被访问 |
| 后端WebSocket | 8080 | WebSocket通信端口 |
| 数据库 | 3306 | 未运行（演示模式） |

### 配置修改

#### 1. 前端配置 (frontend/vite.config.ts)
```typescript
server: {
  port: 5000,           // 改为5000端口（沙箱预览端口）
  host: true,
  proxy: {
    '/ws': {
      target: 'ws://localhost:8080',  // 代理WebSocket请求到8080端口
      ws: true
    }
  }
}
```

#### 2. 后端配置 (proxy-server/.env)
```env
WS_PORT=8080  # 改为8080端口
```

#### 3. 前端WebSocket URL (frontend/src/views/Dashboard.vue)
```typescript
const wsUrl = import.meta.env.VITE_WS_URL || '/ws';  // 使用相对路径，通过代理访问
```

#### 4. 后端代码修改 (proxy-server/src/index.ts)
- 数据库连接改为可选（支持演示模式）
- 即使数据库连接失败，WebSocket服务器也能正常启动

## 管理命令

### 查看服务状态
```bash
# 检查端口占用
ss -tuln | grep -E "(5000|8080)"

# 检查进程
ps aux | grep -E "(vite|node.*dist/index.js)"
```

### 查看日志
```bash
# 前端日志
tail -f logs/frontend.log

# 后端日志
tail -f logs/backend-new.log
```

### 重启服务
```bash
# 停止所有服务
pkill -f "vite"
pkill -f "node.*dist/index.js"

# 启动后端
cd proxy-server && nohup node dist/index.js > ../logs/backend-new.log 2>&1 &

# 启动前端
cd frontend && nohup pnpm dev > ../logs/frontend.log 2>&1 &
```

## 沙箱预览特性

### 支持的功能
1. **设备管理界面**: 可以查看设备列表和控制面板
2. **WebSocket连接**: 可以与后端建立WebSocket连接
3. **实时通信**: 支持消息收发和心跳机制
4. **响应式UI**: 适配不同屏幕尺寸

### 不支持的功能（演示模式）
1. **真实设备数据**: 没有连接真实的TCP设备
2. **历史数据**: 没有数据库存储历史数据
3. **告警记录**: 没有告警功能
